# 新功能测试指南

## 修改内容总结

### 1. 运行状态指示器
- 点击"开始分析"按钮后，表单会显示 loading 遮罩层
- 按钮文字变为"分析中..."或"生成中..."
- 表单内容在运行期间无法交互（透明度降低 + 禁止点击）

### 2. 响应式布局
- 主容器高度固定为 `100vh`，占满浏览器窗口高度
- 侧边栏和主内容区域自适应高度
- 表单内容区域使用独立滚动，避免长表单导致整个页面滚动

## 测试步骤

### 1. 启动前端开发服务器

```bash
cd src/gseda/server/frontend
npm run dev
```

### 2. 测试响应式布局

1. 打开浏览器访问开发服务器地址（通常是 http://localhost:5173）
2. 调整浏览器窗口大小，观察：
   - 窗口变窄时，侧边栏宽度保持不变（280px）
   - 主内容区域自适应剩余宽度
   - 窗口高度变化时，内容区域高度自动调整
   - 表单内容超出时，表单区域内滚动，而不是整个页面滚动

### 3. 测试运行状态指示

1. 选择一个工具（如 "bam-basic-stat"）
2. 填写必要的表单字段
3. 点击"开始分析"按钮
4. 观察以下效果：
   - **按钮状态**：文字变为"分析中..."，显示加载图标
   - **表单状态**：整个表单区域变淡（透明度 60%），无法点击任何控件
   - **遮罩层**：表单中央显示"分析中..."文字
5. 等待工具执行完成
6. 验证：
   - 表单恢复原始状态
   - 结果显示在下方

### 4. 测试多个工具

分别测试以下工具，验证 loading 效果：
- **bam-basic-stat**: 显示"分析中..."
- **reads-quality-stats-v3**: 显示"分析中..."
- **msa-view**: 显示"生成中..."

### 5. 测试动态表单

1. 选择一个不在表单覆盖列表中的工具（如 `fastx-ana` 或其他）
2. 验证动态表单的 loading 效果正常工作

## 验证要点

| 功能 | 预期效果 |
|------|---------|
| 按钮点击 | 立即显示 loading 状态 |
| 表单交互 | loading 期间无法修改表单 |
| 结果展示 | 加载完成后显示执行结果 |
| 响应式 | 窗口大小改变时布局自适应 |
| 滚动行为 | 表单区域内滚动而非页面滚动 |

## 常见问题排查

### 如果 loading 状态不显示：
1. 检查浏览器控制台是否有 JavaScript 错误
2. 检查 `isExecuting` 状态是否正确传递

### 如果布局不响应窗口大小：
1. 确认浏览器窗口高度可见（检查是否有开发者工具占用空间）
2. 检查 CSS 类名是否正确应用到元素

### 如果表单无法滚动：
1. 检查表单内容是否超过容器高度
2. 验证 `.form-container` 的 `overflow-y: auto` 样式已应用
